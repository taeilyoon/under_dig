# Alpha-2-10: 아이템 시스템 설계 (Item System Design)

## 목표
- 게임 내 아이템의 종류, 효과, 사용 규칙 및 인벤토리 구조를 정의합니다.
- 플레이어의 전략적 선택지를 넓히고 생존 가능성을 높이는 시스템을 구축합니다.

## 아이템 분류 (Item Taxonomy)
1. **소모품 (Consumables)**
   - **포션 (Potion)**: 사용 즉시 효과가 나타나는 아이템 (예: HP 회복, 즉시 공격).
   - **주문서 (Scroll)**: 일정 턴 동안 유지되는 버프나 특수 효과를 부여 (예: 공격력 증가, 보호막).
2. **유물 (Artifacts)**
   - 획득 시 패시브 효과를 부여하거나 영구적인 능력치 향상을 제공 (알파 단계 이후 확장 예정).

## 상세 효과 (Item Effects)
- `hp_restore`: 플레이어의 현재 체력을 특정 수치만큼 회복.
- `atk_boost`: 다음 N턴 동안 플레이어의 공격력을 X만큼 증가.
- `def_shield`: 다음 1회의 피해를 무효화하는 보호막 생성.
- `area_damage`: 플레이어 주변 8칸의 모든 적에게 피해를 입힘.

## 인벤토리 구조 (Inventory)
- **슬롯 제한**: 기본 3개~5개의 아이템 슬롯 제공.
- **중첩 여부**: 동일한 아이템 ID를 가진 경우 슬롯 하나에 중첩 가능 (최대 중첩 수 정의 필요).
- **사용 규칙**: 
  - 플레이어의 턴 중 언제든 사용 가능.
  - 아이템 사용은 '빠른 행동'으로 간주되어 턴을 소모하지 않음 (전략적 연계 장려).

## 데이터 모델 스펙
- `Item`: 아이템 기본 정보 (id, name, type, description).
- `ItemEffect`: 구체적인 수치와 지속 시간을 포함하는 효과 객체.
- `Inventory`: 아이템 목록을 관리하고 슬롯 제한을 체크하는 관리자 객체.
