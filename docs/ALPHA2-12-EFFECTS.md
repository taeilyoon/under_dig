# Alpha-2-12: 아이템 효과 시스템 구현 (Item Effects Implementation)

## 목표
- 포션 및 주문서와 같은 소비성 아이템을 사용했을 때 발생하는 실제 게임 로직(HP 회복, 공격력 버프 등)을 구현합니다.
- 지속 시간이 있는 효과(버프/디버프)를 관리하는 시스템을 구축합니다.

## 핵심 효과 구현 (Core Effects)
1. **즉시 회복 (`hp_restore`)**
   - 사용 시 플레이어의 현재 체력을 즉시 증가시킵니다.
   - 최대 체력을 초과할 수 없습니다.
2. **공격력 강화 (`atk_boost`)**
   - 일정 턴 동안 플레이어의 기본 공격력을 추가 수치만큼 증가시킵니다.
3. **방어 보호막 (`def_shield`)**
   - 횟수 기반 또는 턴 기반 보호막을 생성하여 다음 피해를 무효화하거나 경감합니다.

## 효과 관리 시스템 (Effect Manager)
- `Buff`: 지속 시간과 효과 타입을 가진 객체입니다.
- `BuffTracker`: 플레이어에게 걸린 모든 버프를 추적하고, 매 턴마다 지속 시간을 갱신(감소)합니다.
- 지속 시간이 0이 된 버프는 자동으로 제거되고 효과가 소멸됩니다.

## 구현 계획
- `lib/item/item_logic.dart`: 아이템 사용 시 효과를 해석하고 발동시키는 메인 핸들러입니다.
- `lib/models/buff_model.dart`: 버프 데이터 구조를 정의합니다.
- `lib/managers/buff_manager.dart`: 플레이어의 버프 상태를 관리하는 싱글톤 객체입니다.
